## 1. State of the system
A Claude Code multi-agent system that writes a verified Chinese post (AI-news OR з”ҹжҙ»зҫҺеӯҰ) one unit
at a time, proving each unit against a machine-checkable oracle before advancing. This session
reworked it toward **"one skeleton, N fillings"**: the pipeline is fixed; a `mode`/`track` selects
the fillings.

Pipeline (both tracks share the skeleton; gates differ by track):
`S0 decompose + track router вҶ’ S1 scout вҶ’ вҸё gate вҶ’ S2 editorial вҶ’ (grounding 1вҶ’2) вҶ’ S3 write
(вҶ’ fact-check вҶ’ grounding 2вҶ’3, factual only) вҶ’ S4 oracle (HARD) вҶ’ S5 humanize вҶ’ S5.5 Gemini polish
вҶ’ S6 editorial-review (advisory) вҶ’ вҸё human sign-off вҶ’ S7 output`.

**The key structural change вҖ” S1/S3 are mode-agnostic shells + a Read-by-path mode-skill:**
- `.claude/agents/scout.md` (S1) and `.claude/agents/writer.md` (S3) carry NO fixed workflow. The
  orchestrator dispatches them with a `mode` and the exact path of the mode-skill to Read + follow.
- Mode-skills live in `.claude/skills/` (`tech-news-scout`, `aesthetic-scout`, `tech-news-writing`,
  `aesthetic-writing`), each two-part: **workflow + output-contract schema**. All are
  `disable-model-invocation: true` вҶ’ zero context load; consumed by Read-by-path (deterministic,
  NOT description-matching вҖ” the point is predictability).
- `scout` tool whitelist = union incl WebSearch, treated as a **capability ceiling**; behaviour is
  set by the skill workflow (aesthetic-scout has the tool but no retrieval step вҶ’ `mood_pack.json`).
- Adding a content track = write its 2 skills + one row in `run_oracle.py` + an oracle; the shells
  and the playbook do NOT change.

Oracles (the spine) + the dispatcher:
- `tools/citation_audit.py` вҖ” factual: marker existence / freshness / coverage / banned phrases /
  source authority. exit 0/1. CJK-aware.
- `tools/aesthetic_audit.py` вҖ” aesthetic: з ҙжҠҳеҸ· / card length / гҖҢгҖҚ closure / 0X-0N numbering /
  quote-verification (provenance required). exit 0/1.
- `tools/grounding_gate.py` вҖ” faithfulness (1вҶ’2, 2вҶ’3), factual only. exit 0/1.
- `tools/run_oracle.py` вҖ” **NEW** thin per-track dispatcher: `--mode <track>` вҶ’ the one oracle,
  `execv`'d (exit code + stdout pass through verbatim). A switch, not a judge. Aliases:
  `tech_news`/`aesthetic`/`explainer`. Explainer row reserved вҶ’ honest exit 3 (oracle not shipped).

Agents (9): `scout`, `writer` (shells), `editorial`, `fact-checker`, `grounding-checker`,
`citation-auditor`, `humanizer`, `editorial-reviewer`, `output`. (`research` + `aesthetic-writer`
were REMOVED вҖ” folded into the shells + skills.)

**Solid:** both live tracks' oracles (tested green + planted-bug red), the dispatcher, S5.5 offline
machinery, the shell+skill wiring (static-verified). **Partial/unproven:** the reworked pipeline has
NOT been run end-to-end since the refactor; the live Gemini API path is untested; explainer track is
deferred.

## 2. What changed this iteration (with WHY)
Three chunks, all 2026-07-01 (see ITERATION_LOG for the tight version):
1. **`run_oracle.py` dispatcher.** WHY: collapse "modeвҶ’which oracle" into one home (needed at S4 and
   S5.5) so a 4th track = one row; keep the oracles independent + deterministic (never wrap them in
   LLM judgement). Merged in PR #23.
2. **`gemini_polish.py` v2 вҶ’ S5.5.** WHY: promote Gemini polish from a forgettable manual last-mile
   to an auto-run, machine-checked, per-mode step the human reviews via diff. Per-mode temperature
   (factual/explainer 0.3, aesthetic 0.85 вҖ” a direct-API script is where a temperature knob exists,
   unlike a subagent). Re-runs the track oracle via the dispatcher on BOTH pre/post artifacts and
   shows the **beforeвҶ’after finding delta** (green-dashboard-trap antidote at the polish stage) + an
   HTML diff with per-hunk рҹҹўрҹҹЎрҹ”ҙ hints. Aesthetic polishes only non-quote card texts (verified
   quotes keep provenance). `--polished` offline path regenerates the diff without an API call.
   Wired into orchestrator (S5.5, never auto-applies вҖ” diff goes to S6 human gate); STATE `track`
   gains `gemini_polish` + `gemini_temperature`. Merged in PR #23.
3. **Skeleton unification (S1/S3 вҶ’ shell + mode-skill).** WHY above (В§1). Removed `research.md` +
   `aesthetic-writer.md`; added `scout.md`; hollowed `writer.md`; 4 new skills; rewired orchestrator,
   `write-section` / `write-aesthetic-post`, RUNBOOK. This handoff's PR.

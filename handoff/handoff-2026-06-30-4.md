## Design judgments I made (and would defend)
- **Machine vs taste, always split the same way.** Every feature put the deterministic part
  in a Python gate and the judgment part in an agent/human. Caption hook = agent; caption
  *numbers* = `--check-caption`. Source *tier* = `citation_audit`; is-this-article-reliable
  = fact-checker. This is the project's hybrid funnel; I just kept applying it.
- **Allowlist > blacklist** for source authority, and **unknown → WARN, never FAIL** (the
  list can't be exhaustive; hard-failing unknown domains kills legitimate small sources).
- **factcheck_gate fails CLOSED** (empty verdict = FAIL), unlike grounding_gate which
  fails open. Rationale: CLAUDE.md says never green without the fact-check having run, so an
  empty fact-check must not read as pass.
- **Turn fuzzy properties into asserted numbers.** "Does a card overflow?" became
  `max_body_chars <= 430` in the manifest, CI-gated. If you want to hold a property, make it
  a number you can assert — else it regresses.
- **temperature=0 is a red herring in this harness.** Claude Code subagents expose no
  temperature knob, so the real lever for judge reliability is structured-output → Python
  gate. That's exactly what factcheck_gate is; don't chase temperature.

## Traps I hit (so you don't)
- **Short brand tokens over-match.** The `news_discover --min-tier` name classifier first
  tagged *Axios* as tier-1 because `x.ai`'s brand token `"x"` is a substring of "a**x**ios".
  Fix: require brand length ≥4 and stoplist generic subdomain labels (`blog`/`research`).
  Always unit-test a matcher on real names before trusting it.
- **`source .env` clobbered PATH.** Loading the whole `.env` broke `tr`/`sleep` in the
  shell. Parse out only the one key you need; never `source` a secrets file.
- **Committed to `main` by accident once** (#14) — forgot to branch. Recovered with
  `git branch <name> HEAD` then `git reset --hard origin/main`. Branch FIRST.
- **`| tail` hides exit codes.** `python … | tail -1; echo $?` reports tail's status, not
  the command's. Check exit codes without a pipe.
- **One checkout, two agents = churn.** This handoff itself got committed-then-bounced when
  a second agent's `git add -A` swept it in, then correctly backed it out. If two agents
  share a working copy, coordinate branches or you will race.

## What I deliberately left open
- `--source-authority` is opt-in; it is NOT wired into `audit_article.py` by default. Wiring
  it (and the blacklist of real content farms, currently a seed) is a clean follow-up.
- `outline.md` ↔ `outline.json` dual-maintenance — `outline.json` is the source of truth;
  drop `outline.md` or generate it if the duplication bites.
- **No real end-to-end article run yet.** The new per-stage result filenames, the humanized
  artifact flow, and the gates are unit-tested with fixtures but have not been exercised by
  an actual `/write-article` run. That run is the highest-value next step — it will surface
  the next round of findings the way the Cursor review surfaced this one.

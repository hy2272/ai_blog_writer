# Handoff — aiblog multi-agent system — 2026-06-30 (iteration 1)

For the next architect iterating this system. Self-contained. Read `CLAUDE.md` +
`.claude/orchestrator.md` first, then this.

## 1. State of the system
A Claude Code multi-agent system that writes a verified Chinese AI-hot-topic blog/小红书
post one section at a time, proving each section against a written contract AND a
machine-checkable fact/citation audit before advancing. Ported from sas2pyspark's
architecture standards.

Pipeline: S0 decompose → S1 research (sub-agent, live web) → ⏸ angle gate → S2 editorial
(contracts) → grounding 1→2 → S3 writer ∥ fact-check → grounding 2→3 → S4 citation audit
(HARD) → S5 humanize → S6 editorial-review (advisory) → S7 output.

Two oracles (the spine):
- `tools/citation_audit.py` — marker existence / freshness / coverage. exit 0/1. CJK-aware.
- `tools/grounding_gate.py` — faithfulness: each downstream item traces to upstream. exit 0/1.

Solid: both oracles (tested green + planted-bug red), 8 agent specs, orchestrator playbook,
contracts, STATE/DECISIONS, behavior-notes channel, 小红书 output mode.
Partial: S5 humanize + S6 review were folded/skipped in the first real run for speed.

## 2. What changed this iteration (the build itself)
- Built the whole scaffold (28→33 files): CLAUDE.md, orchestrator, 8 agents, commands,
  style_patterns, behavior_notes, citation_audit.py, per-article template.
- Added the **faithfulness layer** (Hanfei's idea): `grounding_gate.py` + `grounding-checker`
  agent + two gates (1→2, 2→3) in the orchestrator. WHY: citation_audit only proves a
  citation EXISTS; grounding proves the downstream item DERIVES from upstream — the middle
  of the funnel (automated → faithfulness → taste → human).
- Made the system **cross-lingual**: English source boundary, Chinese output. WHY: AI
  primary sources are English; the deliverable is Chinese; the grounding judge bridges them.
- Added **小红书 output mode** (behavior note) + a paste-ready `.txt` deliverable shape.
- Added this **/handoff** command + `handoff/` folder (system-level, for the next architect).
- Added `style_patterns.md §7` hard-rule checklist (the translation_patterns §14 analog).

## 3. Bugs fixed in tools
- **citation_audit.py sentence splitter was not CJK-aware** — it required whitespace after
  terminators, but Chinese `。` has none, so a whole paragraph counted as one sentence and
  the uncited-claim check was neutered. Fixed `SENT_SPLIT_RE` to split after CJK
  terminators directly. Caught by the planted-bug demo; regression-tested (EN + CN, good→PASS,
  bug→FAIL). This is exactly the kind of divergence the loop exists to catch.

## 5. Open threads / next improvements (ranked)
1. **Image sub-agent** — currently cover is generated (SVG→PNG via qlmanage, SQUARE only;
   3:4 gets cropped) and official-page screenshots are grabbed via the playwright browser
   manually by the orchestrator. A real `image` agent (v3 had one via SERP+Playwright) would
   close this. In-app screenshots remain the human's.
2. **Dispatch the real grounding-checker / fact-checker / humanizer / editorial-reviewer
   sub-agents** — the first run folded/skipped them for speed. A "full-rigor" run validates
   the whole 8-agent chain.
3. **Custom agents only register in their own project** — they were driven from a sibling
   repo via `general-purpose` sub-agents loading the spec files. Opening `ai_blog_writer/`
   as the project makes `/write-article` + named agents work natively.
4. **qlmanage renders SVG square only** — for a 3:4 小红书 cover, render via a headless
   browser screenshot of the SVG instead.

## 1. State of the system

Core unchanged: S0→S7 staged pipeline, orchestrator-only coordination (sub-agents can't
spawn sub-agents), human gates at S1 (angle) + S6 (sign-off). The leverage is still the
**machine-checkable correctness criterion** that replaces the diff oracle.

The verification stack is now **four layers** (all present, all wired):
- **`tools/citation_audit.py`** — marker existence + freshness + word/keyword/must-cite +
  (opt-in) link-resolve + `--source-authority` domain tier. Exit 0/1; `--strict` promotes
  WARN→FAIL. THE hard gate (S4).
- **`tools/grounding_gate.py`** — faithfulness 1→2 (outline⊂sources) and 2→3
  (draft⊂outline), cross-lingual, per-item structured verdict → PASS/FAIL.
- **`tools/factcheck_gate.py`** (#13) — turns the fact-checker's per-claim verdict JSON into
  a machine gate: any non-SUPPORTED claim → exit 1; **fails closed** on an empty verdict.
  The green-dashboard trap, finally machine-enforced.
- **LLM-judge advisory** — `editorial-reviewer` (S6), screening only, never re-deciding what
  the oracles decided.

Solid: the four-layer verification, the orchestrator playbook, the 小红书 output path (now
redesigned — see §2), per-stage result files + `tools/status.py` matrix (#15), `outline.json`
as the machine outline with `tools/outline_ids.py` lint (#16), S0 discovery via
`tools/news_discover.py` (Apify, #9), source-authority tiering (#11), and now a **local
commit guard** enforcing the iteration-log convention (#18).

Partial / by-design-light: the writing sub-agents are a linear workflow (only research +
fact-check genuinely need fan-out); image rendering still leans on a local Chrome and is
slow (§4); a single clean full-rigor run dispatching every named sub-agent hasn't been done.

## 4. Open threads / next improvements (ranked)

1. **A real `image` sub-agent.** Cards render clean now, but Chrome cold-start is slow (each
   card ~30-40s; several time out and are accepted). A batch-render agent (one Chrome, many
   screenshots) could own cover/illustration choices and kill the timeouts.
2. **Render reliability.** Accepted-on-timeout PNGs mean a flaky Chrome could ship a
   half-painted card silently. Add a post-render dimension/size sanity check, or switch to
   Playwright (already a session MCP) for deterministic shots.
3. **`MAX_BODY_CHARS` is type-scale-coupled.** 680 is render-verified for the current fonts;
   a font/padding change clips a card silently. Prefer a render-time height assertion over a
   static char budget.
4. **Guard coverage.** The pre-commit hook is local + opt-in (needs `core.hooksPath`); a
   fresh clone is unguarded until set. Consider a repo bootstrap script, or a CI step that
   also requires the handoff doc (not just the log) for larger PRs.
5. **Full-rigor single-pass run** dispatching every named sub-agent cleanly.
